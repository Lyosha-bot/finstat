# Finstat — Финансовый учёт

Веб-приложение для учёта личных финансов с визуализацией статистики. Позволяет вести детальный учёт доходов и расходов, распределять их по категориям, устанавливать бюджетные лимиты и анализировать финансовые привычки.

## Возможности

### Управление финансами
- **Транзакции** — добавление, редактирование и удаление операций (доходы и расходы)
- **Категории** — системные и пользовательские категории для классификации операций
- **Бюджеты** — установка месячных лимитов по категориям с отслеживанием расхода
- **Фильтрация** — поиск по описанию, фильтр по периоду, типу и категории

### Инфографика
- Доходы и расходы по категориям
- Исполнение бюджетов с цветовой индикацией
- График накопления баланса
- График динамики доходов и расходов по месяцам
- Расходы по дням недели
- Средние значения за период

### Безопасность
- Регистрация и авторизация пользователей
- JWT-токены (access + refresh) с автоматическим обновлением
- Хеширование паролей (bcrypt)
- Защита маршрутов на уровне middleware

### Интерфейс
- Тёмная тема с glassmorphism-эффектами
- Адаптивный дизайн (320px — 1920px)
- CSS Custom Properties для единообразия стилей
- Skeleton loading при загрузке данных
- SVG-графики без сторонних библиотек

## Быстрый старт

### Требования
- Docker и Docker Compose

### Запуск

1. Клонируйте репозиторий:
```bash
git clone https://github.com/Lyosha-bot/finstat.git
cd finstat
```

2. Создайте файл `.env` в корне проекта:
```env
DB_NAME=finstat
DB_USER=finstat
DB_PASSWORD=finstat
DB_HOST=postgres
DB_PORT=5432

BACKEND_PORT=8081
FRONTEND_PORT=5173
WEB_PORT=8080
```

3. Соберите и запустите:
```bash
docker compose build
docker compose up
```

4. Откройте в браузере: **http://localhost:8080**

### Остановка
```bash
docker compose down
```

## API

Полная Swagger-документация доступна по адресу `http://localhost:8080/api/docs/index.html` после запуска.

### Авторизация

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `POST` | `/api/auth/register` | Регистрация нового пользователя |
| `POST` | `/api/auth/register/is-valid` | Проверка данных регистрации |
| `POST` | `/api/auth/login` | Вход в систему (возвращает access-токен) |
| `POST` | `/api/auth/refresh` | Обновление access-токена |
| `POST` | `/api/auth/logout` | Выход из системы |

### Транзакции

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `POST` | `/api/transactions` | Создать транзакцию |
| `GET` | `/api/transactions` | Список транзакций (с фильтрами) |
| `PATCH` | `/api/transactions/:id` | Обновить транзакцию |
| `DELETE` | `/api/transactions/:id` | Удалить транзакцию |

**Параметры GET /api/transactions:**
- `from` — начальная дата (YYYY-MM-DD)
- `to` — конечная дата (YYYY-MM-DD)
- `type` — тип: `1` (доход), `-1` (расход)
- `categories` — список ID категорий (повторяемый параметр)
- `limit` — количество элементов на странице
- `page` — номер страницы

### Категории

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `GET` | `/api/categories` | Список всех категорий |
| `POST` | `/api/categories` | Создать пользовательскую категорию |
| `PATCH` | `/api/categories/:id` | Обновить категорию |
| `DELETE` | `/api/categories/:id` | Удалить категорию |

### Бюджеты

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `POST` | `/api/budgets` | Создать бюджет |
| `GET` | `/api/budgets?date=YYYY-MM-DD` | Список бюджетов с текущими тратами |
| `GET` | `/api/budgets/category/:id?date=YYYY-MM-DD` | Бюджет по категории |
| `PATCH` | `/api/budgets/:id` | Обновить лимит бюджета |
| `DELETE` | `/api/budgets/:id` | Удалить бюджет |

### Аутентификация

Все защищённые эндпоинты требуют заголовок:
```
Authorization: <access_token>
```

Refresh-токен передаётся через HttpOnly cookie `refresh_jwt`.
